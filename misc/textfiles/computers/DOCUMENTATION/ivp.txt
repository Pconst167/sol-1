ÄÄÄ[ ABOUT THE CONFIGURATION FILE ]ÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄ

  Below is a detailed explanation of each option that is available in the
configuration file.  The format of each option is a letter with a equal sign
and the option following.  A sample configuration file is included.

ş F - Specifies the file name that all the output will be written to.
    - Ex. "F=TEST.ASM"
ş V - Specifies the name of the virus.
    - Ex. "V=A Test Virus"
ş A - Specifies the name of the author.
    - Ex. "A=Admiral Bailey"
ş I - Specifies what type of files to go for.
    - C=Com, E=EXE, B = Both (Exe and Com), T=Trojan
    - Ex. "I=B"
ş G - Specifies what to do with the file.
    - O=Overwriting, A=Appending
ş W - Is for the ID code used by EXE infectors.
    - It can only be (and must be) two characters.
    - Ex. "W=AB"
ş E - Specifies if encryption or no encryption is to be used.
    - No encryption reduces the size of the virus.
    - Ex. "E=Y"
ş D - Specifies if you want the virus to change directories.
    - If yes then the directory changing will be done the '..' way.  Where
    - the virus will step up one directory closer to the root each time.
    - Ex. "D=Y"
ş C - Specifies if you want your virus to infect COMMAND.COM.
    - If no then any 8 character file ending in 'ND' will not be infected.
    - Ex. "C=N"
ş B - Specifies if you want the virus to replace INT 24h.  If yes then any
    - writes to a write protected disk will be ingnored and aborted.
    - Ex. "B=Y"
